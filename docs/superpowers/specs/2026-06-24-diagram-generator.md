## 0. CEO/eng/design review outcome (2026-06-25, HOLD SCOPE) — READ FIRST

A plan review (CEO + outside-voice challenge) reshaped this spec before any build. The architecture
(plan-then-render, freeze-to-static, asset-stage integration) survived. Two premises did NOT survive
unchallenged and are now spike-gated:

1. **Render primitive — RESOLVED by working evidence (D2 closed).** Two CEO-built experiment projects
   (2026-06-25) already produced production-looking videos on the SAME content (`coffee-vs-tea`,
   `ai-models-showdown`), and the look that works is **FLAT clean SVG, not sketch** — so **RoughJS is
   dropped, and with it the Node + jsdom toolchain.** Conceptual diagrams are authored as **flat
   parametric SVG inside one pure `renderFrame(ms)` function** (the experiments' exact pattern, which is
   also Mason's frame-seek model). See §0.5. The spike's job narrows to layout/feasibility, not an
   aesthetic bake-off.
2. **"Templates own layout" hid the hard 90%.** RoughJS/SVG only draws shapes you already positioned;
   good-looking deterministic graph layout is a Graphviz-class problem with no design here. **Resolution:
   fixed-slot templates with literal-constant positions only — NO free-form layout engine.**

Settled forks (folded below): drop the 2nd headless-Chrome pass (parse the SVG string in Python; Mason's
gate is the only browser backstop); gate the build on a real diagram-shot frequency count; add a
conservative fallback + an LLM self-check for confidently-wrong-but-valid diagrams; add observability
counters (fallback-to-stock rate, retry rate); name every failure-path rescue (see §14 D9–D14 and the
Failure-Modes registry in the review report at the end of this file).

**The whole approach stays reversible (4/5): `classify_shot` reverts to the stock fallback that already
ships.** Do the spike (§15 step 0) before committing to any of the build.

## 1. Goal & problem
Turn a plain-English `kind:"diagram"` storyboard shot — e.g. *"a chat bubble sprouting small robotic
arms reaching toward buttons"* — into a clean, on-brand, **hand-illustrated-style** inline-SVG visual
that drops into a 1280×720 scene (caption band = bottom ~140px must stay clear), passes the automated
`lint`/`validate`/`inspect` gate (no `text_occluded`, no overflow, no console errors), reliably and
fast, **no human in the loop**, ~10–20 diagrams per video.

Today `kind:"diagram"` conceptual shots fall through to a blank placeholder; a stock-photo stopgap
ships now (`asset-sourcer/source_engine.py` routes them to "source a relevant image"). This is the
real fix.

### Hard constraints (the pipeline discipline — design around these)
- **Renderer = HyperFrames CLI** (`npx hyperframes`, headless Chrome → MP4). Native format is
  self-contained HTML/CSS/**inline SVG**. **No render-time external fetch** (the determinism wall).
- **Determinism wall:** no render-time network; no `Math.random`/`Date.now` in rendered HTML; a given
  storyboard renders **byte-stable**. The self-scan also **blocks SMIL `<animate>`**
  (`composition_engine.py` `_SVG_ANIM_RE`).
- **Closed-vocabulary discipline:** layouts/effects/textures/transitions are closed sets; an unknown
  token is a hard input-validation error, never silently dropped.
- **Frozen-but-extensible contracts:** extend schemas by ADDING optional fields under a bumped
  `schema_version` with `additionalProperties:true`. Old readers keep working.
- **House style:** flat, editorial, minimal. Palette cream `#f2ede4` bg / near-black `#2b2722` ink /
  warm accent `#c8632b`. Fraunces (display) + Inter (body), both already bundled OFL locally.
- **LLM = Claude via Agent SDK on a subscription seam** (no metered key). Small calls (~10s) fine;
  large/complex single generations are unreliable (see §3).
- **Engines pure + injectable, never import Atlas.** Mason is pure deterministic code; any LLM call
  lives behind a `chat_fn` seam and runs in Magpie, off the render path.

## 2. Research basis (verified — confidence high unless noted)
**(R1) Why freeform "one SVG per shot" fails — the 10s→150s blowup is three compounding causes**
(merged 3-0): output-token length is a near-linear latency driver (a full 1280×720 SVG is thousands
of tokens vs dozens for a small shape); **extended thinking spends a SEPARATE thinking-token budget
timed as output**, risking time-outs; and the **Agent SDK can stall ~185s emitting zero events
mid-stream** (n=1). Levers: bounded output, `max_tokens` cap (blunt — truncates), Haiku as the
fastest model, and a **mandatory wall-clock timeout + retry**.
Sources: [SDK issue #44](https://github.com/anthropics/claude-agent-sdk-typescript/issues/44),
[reduce-latency](https://platform.claude.com/docs/en/test-and-evaluate/strengthen-guardrails/reduce-latency),
[extended-thinking](https://platform.claude.com/docs/en/build-with-claude/extended-thinking).

**(R2) RoughJS is the correct hand-drawn / "boiling-line" primitive** (merged 3-0): <9kB, sketchy
hand-drawn style, works in SVG via `rough.svg()` returning `<g>` nodes appendable with
`svg.appendChild(node)`, **no fetches**. Parametric vocabulary (lines, curves, arcs, polygons,
circles, ellipses, paths). **Deterministic via a numeric `seed` (1…2³¹)** — same params + seed →
identical shape; unseeded → `Math.random`. Caveat: **static only** — animated boiling-line would need
per-frame re-seed or an `feTurbulence`+`feDisplacementMap` filter (the latter blocked by our SMIL
rule). Sources: [roughjs.com](https://roughjs.com/), [rough repo](https://github.com/rough-stuff/rough),
[wiki](https://github.com/rough-stuff/rough/wiki).

**(R3) Rough Notation = closed fixed 7-type annotation layer on RoughJS/SVG** (merged 3-0): exactly
underline, box, circle, highlight, strike-through, crossed-off, brackets — an emphasis layer, not a
freeform generator. Injects an SVG sibling at runtime and **defaults to `Math.random`** → bake the 7
shapes in at asset time; don't ship its runtime lib. Sources:
[roughnotation.com](https://roughnotation.com/), [repo](https://github.com/rough-stuff/rough-notation).

**(R4) Plan-then-render is the literature consensus** (merged 3-0): DiagrammerGPT (COLM 2024) uses an
LLM planner-auditor loop to produce/refine a diagram **plan**, then a separate generator + text-label
module renders it — because end-to-end image models lack fine-grained layout control on
densely-connected arrows/lines and fail to render legible labels. Chat2SVG (CVPR 2025) has the LLM
emit only basic geometric primitives. **Adopt plan-first with a DETERMINISTIC render** (not Chat2SVG's
diffusion+optimization). Sources: [arxiv 2310.12128](https://arxiv.org/pdf/2310.12128),
[chat2svg.github.io](https://chat2svg.github.io/).

**(R5) Generative image models are a viable FALLBACK, pre-rendered once** (merged 3-0): Recraft V3
exposes hand-drawn/sketch styles by name (`hand_drawn`, `hand_drawn_outline`) + a reusable custom
`style_id` locking one house style; Ideogram saves a custom style from ≤3 reference images. Generate
once at the asset stage, download raster, render stays offline. **Caveats:** metered APIs OUTSIDE the
subscription seam; raster won't recolor cleanly to the exact palette; composite into the top ~580px to
leave the caption band empty. Sources: [Recraft styles](https://www.recraft.ai/docs/api-reference/styles),
[Ideogram style-reference](https://docs.ideogram.ai/using-ideogram/features-and-tools/reference-features/style-reference).

**(R6) Integration at the ASSET stage** (medium — architectural, confirmed against code): because any
LLM call can stall/retry, run plan→render→validate **off the render critical path** in Magpie; Mason
composes a frozen, validated asset deterministically.

**Refuted (do NOT rely on):** the sketch aesthetic is "fundamentally random" (0-3 — it's seedable);
`display:"omitted"` speeds first-token (1-2); `budget_tokens < max_tokens` gives a clean latency cap
(0-3); output-length-alone is the stall trigger (1-2 — it's the three causes together).

**Caveats:** Haiku model id is version-sensitive (re-verify). The 185s stall is n=1, so timeout+retry
is mandatory. Seed-determinism assumes a platform-independent PRNG — **moot here because we freeze to
static SVG at asset time** (§4).

## 3. The core decision
**Parametric plan-then-render, with RoughJS rendered to a FROZEN STATIC SVG at the ASSET stage.**
The LLM never draws an SVG — it emits a tiny structured **DiagramPlan** (closed component vocabulary +
labels + coarse layout hints). Deterministic Node code renders that plan through RoughJS to a **static
`.svg` file** written locally by Magpie. **Mason embeds the frozen SVG inline** exactly like it already
inlines brand-chip SVGs.

**The load-bearing insight — freeze to static SVG:** because RoughJS runs in Magpie's Node at asset
time and the output is a frozen file, the RENDER stage never executes RoughJS. Two hard problems vanish
*by construction*: (a) **no `Math.random` in the rendered scene**, (b) **byte-stable output** — the
file is written once and copied verbatim, like the bundled fonts and brand-chip SVGs already are. It
also collapses the "does seeded RoughJS reproduce byte-identical on the HyperFrames Chrome build?"
open question entirely (the PRNG runs in Magpie, not in render Chrome).

## 3.5. LOCKED architecture (Option ii — animated flat SVG composed by Mason) [D16]

Supersedes the static-SVG/Node flow in §3–§5 (incl. the §4 data-flow diagram and the §5 DiagramPlan's
implication of a rendered file) wherever they conflict. The split:

- **PLAN — Magpie (asset stage, cached per `shot_id` per D13).** Haiku emits the DiagramPlan (closed
  component + **animation-technique** tokens + labels + coarse layout hint). Validated (closed-vocab +
  semantic self-check D11) and stored in `asset_manifest`. **No rendering, no Node, no SVG file** — just
  the cached plan.
- **COMPOSE — Mason (compose stage, pure Python).** Reads the cached plan and builds **animated flat SVG
  inline + GSAP timeline tweens** from a CLOSED technique library, on the scene's paused master timeline
  (`window.__timelines["scene-NN"]`) — exactly how Mason already builds `render_bar_chart` and emits
  effect tweens. Passes the existing lint→validate→inspect gate.

**Closed technique tokens** (ported from `claude experiments`; extend deliberately, unknown = error):
static shapes {labeled-box, container, speech-bubble, layer-stack, before→after, cycle, grid,
fixed-slot node+edge, glyph set} + **animations** {draw-on (`stroke-dashoffset`), count-up, grow-bar,
riding-dot (`getPointAtLength`), cross-fade, seeded-particles}. Glyphs are pre-authored SVG path
constants in the library (like `BRAND_CHIPS` `logo_svg`).

**Determinism:** no Node/jsdom, no static-SVG file. Motion rides Mason's existing **build-time GSAP
timeline** (`.from/.to`, no SMIL `<animate>`, no `Date.now`/`Math.random`); any "random" value (e.g.
particle layout) is a **seeded constant baked at compose** from `hash(shot_id)`. Byte-stable because the
plan is cached (D13) and Mason is deterministic.

**This DROPS** vs the original §3–§5: the frozen-static-SVG handoff (D3 superseded), the
Node/RoughJS/jsdom toolchain (D2), the asset-stage headless-Chrome pass (D10), and the cross-language
seam. Net: a small cached LLM plan in Magpie + pure-Python animated-SVG composition in Mason.

```

   │

   │  1. PLAN     Haiku 4.5 via chat_fn → DiagramPlan JSON
```

## 5. The DiagramPlan contract (LLM output — concrete shape)
Tiny, terse, closed-vocab JSON. The LLM fills this; it never positions pixels (templates do, §7).

```jsonc
{
  "canvas": { "w": 1280, "h": 720, "safe_bottom": 140 },   // renderer-supplied, not LLM
  "layout_hint": "left-to-right",   // enum: left-to-right | radial | stacked | grid | freeform
  "components": [
    {
      "type": "speech-bubble",      // CLOSED vocab (§6) — unknown = validation error
      "label": "AI",                // short text; renderer keeps it out of the safe-bottom band
      "of": "robot-arm",            // optional: glyph the component carries/emits
      "to": ["button", "button"],   // optional: targets for arrows/edges (by id or glyph)
      "emphasis": "circle"          // optional: Rough Notation 7-type, baked static
    }
  ]
}
```
- **Validation at the seam:** every `type`/`emphasis`/`layout_hint` must be in the closed set →
  unknown = error (retry the plan, then fall back). No coordinates, no SVG, no styling in the plan.
- Keep it terse: short keys, enums, no prose. Terseness (not the cap) is what keeps the call fast.

## 6. Closed component vocabulary (v1 — validate ~90% coverage during build)
Unknown token = error (mirrors the engine's existing closed vocabularies). ~12 components:

| Component | Purpose | Layout owned by renderer |
|---|---|---|
| `speech-bubble` / `thought-bubble` | dialogue / cognition | tail auto-placed, label centered |
| `arrow` (straight/curved/loop) | flow / causation | routed between anchors |
| `node+edge` (flow/graph) | relationships | force-directed / layered |
| `labeled-box` / `container` | grouping | auto-sized to label |
| `layer-stack` | layers / hierarchy | vertical offset stack |
| `before→after` | transformation | two panels + connector |
| `cycle` / `loop` | repetition / feedback | circular arrow ring |
| `grid` / `matrix` | comparison / 2×2 | even cells |
| `annotation` | emphasis (Rough Notation 7) | overlaid on target |
| `glyph` (closed set) | concrete icon | placed per parent |

**Closed glyph set (v1):** person, gear, document, robot-arm, button, brain, cloud, database, lock.
Example: *"chat bubble sprouting robotic arms reaching toward buttons"* →
`{speech-bubble(of:robot-arm, to:[button,button,button])}` + arrows. A composition, not a drawing.

## 7. House style, layout ownership & determinism
- **Style hardcoded in the template library, never LLM-chosen:** `stroke` width, RoughJS `roughness`,
  `bowing`, `fillStyle` (e.g. `hachure`), palette (`#2b2722` ink, `#c8632b` accent, `#f2ede4` bg),
  Fraunces/Inter for labels. → every diagram across a video is identical house-style **by
  construction**.
- **Seed = `hash(shot_id)`** → reproducible, varies per shot, never random.
- **Templates own layout (D8):** the renderer positions everything — column flow, force-directed
  graph, radial cycle, **and reserves the bottom 140px** (nothing is ever placed there → caption-band
  clearance is pass-by-construction). This is what makes the small-model choice low-stakes (the LLM
  never does spatial placement).
- **Determinism:** frozen static SVG → no render-time JS, no `Math.random`, no network; byte-stable.
  No SMIL `<animate>`. No stray `<title>`/`<text>` in the caption band (reuse the brand-chip
  `text_occluded` lesson — Mason already strips SVG `<title>`).

## 8. LLM call configuration (the plan step)
- **Model:** Haiku 4.5 (`claude-haiku-4-5`) by default, behind the `chat_fn` seam (swappable per call).
  Rationale: the task is constrained composition→JSON (extraction/classification), Haiku's wheelhouse,
  and Haiku is itself the speed lever. **Escalation:** gate on a 10–20 real-shot eval (plan validity +
  does the plan match the description); escalate the plan step to Sonnet if Haiku's component/layout
  picks miss; optionally complexity-route later (Haiku simple / Sonnet dense graphs). [D6]
- **`max_tokens` = 2000.** A truncation guard, NOT a target — latency scales with tokens *actually
  emitted*, not the cap, so a generous ceiling is free insurance. **Escalation clause: if dense
  diagrams truncate during build (`stop_reason == "max_tokens"` or invalid JSON), ASK THE CEO — we can
  raise the cap.** Treat truncation as a retry-with-higher-cap signal in the meantime. [D5]
- **No extended thinking** on this call — it spends a separate budget that can truncate the JSON before
  any output and inflate latency. [D7]
- **Wall-clock timeout ~20s + retry ×1** — the ~185s SDK stall is the real risk. [from R1]
- **Structured output:** force the DiagramPlan schema (tool/structured-output), so the model returns
  validated JSON, not prose.

## 9. Validation loop (off the render path, in Magpie)
Load the standalone `.svg` in headless Chrome (same discipline as the gate) and assert:
1. **No overflow / no console errors.**
2. **Caption-band clearance:** no element bbox enters the bottom 140px (already pass-by-construction
   via the reserved band; validator is the backstop).
3. **Not empty/garbled:** rendered ink bbox is non-trivial AND element count ≈ the plan's component
   count.
On fail: **re-prompt the plan once** (feed the failure reason) → **relevant stock image** (existing
Magpie path) → **styled placeholder**. Mason's `run_gate` (lint→validate→inspect) stays the final
backstop at render time.

## 12. How it fits d3 + GSAP + Lottie (the motion-stack spec)
- **d3** = the data path (`data-chart` layout). Diagrams are the non-data conceptual path d3 doesn't
  cover — complementary, no collision.
- **GSAP** = the diagram's draw-on / boiling reveal as a NEW closed EFFECT token `sketch-draw-on`
  (via `stroke-dashoffset` on the paused master timeline — same mechanism as the existing `map-draw`).
  This is exactly the motion-stack spec's **Phase M1** (deeper GSAP as new effect tokens). Static SVG
  v1 ships without it; add the reveal as a closed token.
- **Lottie** = premium sourced/licensed motion. RoughJS static SVG is the lighter, in-house,
  zero-licensing, fully-deterministic DEFAULT; Lottie is the upgrade for genuine rich motion.
- The diagram generator is the conceptual-illustration sibling of the spec's "Loop 🎞️" generator —
  fold into Magpie, or make it an off-pipeline agent of the same fleet shape.

## 13. Latency / cost / quality trade-offs
| | Freeform SVG (rejected) | **Parametric plan→render (primary)** | Image-gen fallback |
|---|---|---|---|
| Per-shot latency | unbounded (>150s hangs) ON render path | Haiku ~2–5s, capped 20s + retry; render ~ms; OFF render path, parallel | seconds, pre-rendered once |
| Reliability | one hang stalls the whole render | timeout+retry bounded; degrades to stock | metered API can fail |
| Determinism | render-time JS / Math.random risk | **byte-stable frozen SVG, gate-safe** | static raster, fine |
| Style consistency | drifts per generation | **identical by construction** | locked `style_id`, off-palette risk |
| Cost | subscription seam | subscription seam | **metered, outside the seam** |
